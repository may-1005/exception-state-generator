## 3. 分析任务
请扫描这个设计稿，完成以下三个步骤：

### 步骤一：组件识别
- 列出界面中所有可交互和展示类组件（例如：列表、卡片、按钮、图表、输入框、头像区、标签栏等）
- 对每个组件，简要描述其预期内容（例如：列表展示用户评论、图表展示销售数据）

### 步骤二：状态缺口分析
对照以下清单，为**每个组件**判断哪些状态在设计稿中**缺失**：

| 状态类型 | 典型场景 | 检查项 |
| :--- | :--- | :--- |
| 空状态 | 无数据、无搜索结果、首次进入 | □ 缺失 |
| 错误状态 | 加载失败、提交失败、请求超时 | □ 缺失 |
| 加载中 | 骨架屏、loading 动画、进度指示 | □ 缺失 |
| 无权限 | 未登录、权限不足、内容被屏蔽 | □ 缺失 |
| 网络异常 | 断网提示、重试按钮 | □ 缺失 |
| 数据极多 | 滚动截断、查看更多、分页/虚拟滚动 | □ 缺失 |
| 数据极少 | 单条记录展示、空白区域的视觉平衡 | □ 缺失 |
| 编辑/操作反馈 | 成功提示、删除确认、撤销入口 | □ 缺失 |

### 步骤三：生成优先级建议
- 高优先级（影响核心功能闭环，如登录后无数据、提交失败）
- 中优先级（体验降级但不阻断流程，如搜索无结果）
- 低优先级（边缘场景，如超长用户名截断）

## 5. 额外要求
- 对每个缺失状态，给出**一句话的具体视觉/交互建议**（不要只写“需要空状态”，而是写“空状态应居中显示占位图 + 标题+引导按钮”）。
- 如果设计稿已包含某个状态，请在“已有状态”列中注明，不必重复建议。
