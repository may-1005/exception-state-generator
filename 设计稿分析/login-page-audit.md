# 登录页设计稿状态缺口分析

> 分析对象：视频联网整合平台登录页
> 规范依据：CLAUDE.md（缺省规范 + 错误提示规范）

## 步骤一：组件识别

| # | 组件 | 类型 | 预期内容 |
| :--- | :--- | :--- | :--- |
| 1 | 平台标题/Logo | 展示类 | 展示"视频联网整合平台"名称及英文副标题 |
| 2 | 用户名输入框 | 交互类 | 接收用户输入的登录账号 |
| 3 | 密码输入框 | 交互类 | 接收用户输入的登录密码 |
| 4 | 登录按钮 | 交互类 | 提交登录表单、触发登录请求 |
| 5 | 版本号 | 展示类 | 显示当前平台版本（V2.0.0） |
| 6 | 背景区域 | 展示类 | 全屏品牌背景图，营造科技感氛围 |
| 7 | 整体登录表单 | 容器类 | 承载上述组件的登录卡片/区域 |

## 步骤二：状态缺口分析

| 组件名称 | 已有状态 | 缺失状态 | 补充建议 |
| :--- | :--- | :--- | :--- |
| 平台标题/Logo | 正常展示 | 无明显缺失 | 纯展示组件，不涉及状态切换，现有设计已满足需求 |
| 用户名输入框 | 默认态（含 placeholder "用户名"） | 🟡获取焦点态<br>🟡输入中状态<br>🔴校验错误态<br>🟡禁用态 | 校验错误时输入框边框变红，下方出现红色提示文案"请输入用户名"；焦点态边框高亮为主题蓝色 |
| 密码输入框 | 默认态（含 placeholder "密码"） | 🟡获取焦点态<br>🔴校验错误态<br>🟡显示/隐藏切换<br>🟡禁用态 | 校验错误时输入框边框变红，下方提示"请输入密码"；右侧增加眼睛图标用于切换密码明文/密文显示 |
| 登录按钮 | 默认态 | 🔴加载中状态<br>🟡hover 态<br>🟡按下态<br>🟡禁用态 | 点击后按钮显示 loading 动画 + "登录中..."文案，同时置灰不可点击，防止重复提交；未填写账号密码时按钮置灰 |
| 版本号 | 正常展示 | 无明显缺失 | 纯展示元素，现有设计已满足需求 |
| 背景区域 | 背景图正常展示 | 🟢背景图加载失败 | 兜底使用深蓝渐变背景色（如 `#0a1a3a → #0d2b5e`），避免背景图加载失败时显示白屏 |
| 整体登录表单 | 正常展示（默认态） | 🔴登录失败轻提示（Toast）<br>🔴网络异常提示<br>🔴服务器异常提示<br>🟡无权限阻断提示（Dialog）<br>🟢忘记密码入口<br>🟢记住密码选项 | 登录失败用 Toast 提示具体原因（"用户名或密码错误"/"账号已被锁定"），避免使用技术术语；网络异常 Toast 提示"网络连接失败，请检查网络后重试"并保留已填写内容；服务器异常用 Dialog 阻断并引导"稍后重试" |

## 步骤三：优先级汇总

### 🔴 高优先级（影响核心功能闭环）
- 登录按钮加载中状态 — 防止重复提交，直接影响登录成功率
- 登录失败 Toast 提示 — 用户需要知道为什么登不上去
- 输入框校验错误态 — 表单提交的基础反馈，缺少会导致用户困惑
- 网络异常/服务器异常提示 — 非正常流程的兜底反馈

### 🟡 中优先级（体验降级但不阻断流程）
- 输入框焦点态/hover 态/按下态 — 交互反馈的完整性
- 密码显示/隐藏切换 — 常见登录体验，减少密码输入错误
- 按钮禁用态 — 表单为空时引导用户先填写
- 无权限阻断提示（Dialog） — 明确告知无权限原因及处理方式

### 🟢 低优先级（边缘场景）
- 背景图加载失败兜底 — 极少数网络环境下图片加载失败的保底
- 忘记密码入口 — 拓展流程，非核心闭环
- 记住密码选项 — 便捷功能，非必须
