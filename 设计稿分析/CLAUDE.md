# 设计规范与AI行为规则

## 基础规则
- 始终用中文回复。
- 输出格式优先使用 Markdown 表格。

## 界面分析要求
请为页面（全局）及每个组件（局部）检查以下状态：

### 缺省状态（参考内部缺省规范）
- **数据为空**：是否有空状态插图/图标 + 引导文案 + 可选操作按钮
- **无权限**：是否有权限标识 + 说明文案 + 申请/返回操作
- **内容已删除/下架**：访问不存在内容时是否有相应提示
- **搜索无结果**：搜索/筛选无匹配时是否有空状态指引
- **网络异常**：网络断开时是否有离线提示
- **服务器异常**：服务端报错时是否有兜底提示
- **初次使用引导**：首次进入功能模块是否有新手引导
- **数据加载中**：是否有骨架屏/loading 加载态

### 错误提示规范（参考内部错误提示规范）
- **轻提示 Toast**：操作结果是否有轻量反馈，自动消失
- **对话框 Dialog**：需用户确认的阻断式错误是否正确使用
- **表单校验**：行内校验错误是否标红字段 + 错误文案
- **页面级错误**：404/500 等整页错误是否覆盖
- **通知 Notification**：全局持续通知是否正确使用
- **错误文案**：是否说明了原因 + 引导操作，避免技术术语

## 输出要求
- 对每个缺失状态，给出具体的视觉/交互建议（不要只说"需要XX"）。
- 优先级标注：🔴高 🟡中 🟢低。
